Overview

Built a service platform enabling users to submit breakdown requests and automatically notify the nearest available mechanic/tow service. Implemented request assignment, live status, and feedback collection. Focused on reliability and clear UI for time-critical interactions.

Objectives

Reduce response time for roadside assistance; ensure consistent experience on mobile; and validate API workflows thoroughly.

Outcomes

Validated core flows with manual + API tests; improved user confidence with accurate ETA/status updates.

Key Features

  • Live status and ETA updates
  • Simple request and feedback flows
